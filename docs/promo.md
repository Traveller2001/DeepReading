# 做了一个论文精读工具，让 LLM 像人一样"翻着读"论文

去年 Deep Research 方向挺火的，各家都在做能自动搜索大量文献、归纳整理成报告的 agent。我自己也用了不少，确实能帮你快速了解一个领域的全貌。但用多了会发现一个问题：这类报告谈的往往比较泛，每篇文献点到为止，真正感兴趣的论文读完报告之后其实还是想自己再专门精读一遍。

所以我就想，与其追求广度，不如先把"读好一篇"这件事做扎实。另外我发现这些工具生成的报告基本都是纯文字的，但很多技术没有图文结合其实很难理解——比如一个模型的架构、一个多阶段的训练流程，光靠文字描述读起来还是费劲。我希望生成的报告里也是带图的。

基于这些想法，做了 [DeepReading](https://github.com/Traveller2001/DeepReading)，分享一下。

## LLM 是"翻着读"论文的

不是把全文一股脑塞进 context 让模型自由发挥。模型会先调用工具查看论文结构，然后选择性地精读关键页面、搜索特定内容、查看图表上下文。整个过程前端有一个 thinking panel 实时展示模型在做什么——"正在阅读第 3 页"、"正在搜索 attention mechanism"。这些 tool call 通常有十几轮，生成的报告质量和盲读区别还是挺大的。

## 引用是可以点的，而且点得准

报告里的每个关键论述后面都有引用标记，比如 `[p.4]`。点击后右侧 PDF 会自动跳转到对应页面，并高亮引用的原文片段。这个定位不是简单的"跳到第几页"，而是用 pdf.js 的文本图层做了逐字符匹配，精确到行。

做这个功能的时候踩了不少坑。一开始只记录页码，发现一页内容太多根本找不到；后来加了 y 坐标做粗定位，但不同 PDF 的坐标体系不统一；最后还是回到了文本匹配——把引用的原文短语在 PDF 文本层中定位，做多策略 fallback（精确匹配 → 弹性空白匹配 → 前缀匹配），效果才稳定下来。

## 报告带图，而且能自动生成图

论文里的图表会被自动提取出来，报告按需引用原图。同时模型也会判断哪些概念适合用图来辅助理解，然后调用工具用 HTML/SVG 自动生成流程图、架构图之类的示意图。比如一个多阶段的训练 pipeline 或者 MoE 的路由机制，生成一张结构图比三段文字描述清楚得多。

## 自动讨论，然后再改报告

这是我觉得比较有意思的一个设计。报告生成后，系统会自动启动一轮"讨论"——一个 Reader Agent 扮演第一次读报告的外部读者，针对报告中不够清晰的地方提问；一个 Writer Agent 根据论文原文回答。三轮 Q&A 之后，Writer 参考讨论内容对报告进行润色修改。

效果上来说，讨论阶段经常能揪出报告里一些含糊的表述或者遗漏的细节，润色后的版本确实会更完整一些。整个讨论过程在右侧面板实时展示，也可以当作对论文的补充理解来看。
